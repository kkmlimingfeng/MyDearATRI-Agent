# 近期开发计划

## 1. ReAct 循环支持并行工具调用

**现状**：每次循环只能执行一个 Action，无依赖的工具需要多轮串行。  
**目标**：优化提示词和解析逻辑，允许 Agent 在一次循环中输出多个无依赖的 Action，并发执行（最大并发数 5）。  
**涉及文件**：
- `agent/react_agent.py`：解析多个 Action、并发执行、合并多个 Observation
- `modules/prompt/SystemPrompt/AGENT.md`：说明何时可以并行

---

## 2. 记忆系统升级

**现状**：仅实现了基于 `memory.md` 的长期记忆，且未充分验证效果。  
**目标**：
- 短期记忆：明确实现方式（目前 `_history` 可视为短期记忆，但需进一步设计窗口、摘要策略）
- 仿人脑三层记忆：工作记忆 / 短期记忆 / 长期记忆
- 测试长期记忆在上下文中的实际效果

**涉及文件**：
- `modules/mem/memory_module.py`
- `agent/react_agent.py`

---

## 3. RAG 与向量数据库

**现状**：未开发，向量化能力未使用。  
**目标**：引入向量数据库（如 FAISS / Chroma / 轻量级嵌入），实现：
- 记忆、文档、技能的向量化检索
- 长上下文的 RAG 增强

**涉及文件**：
- 新增 `modules/rag/`
- `modules/mem/`
- `agent/react_agent.py`

---

## 4. Reviewer 模块优化

**现状**：每次对话结束后调用一次 LLM 做总结。  
**目标**：
- 减少模型调用次数（长期目标）
- 只在“有价值”的对话后触发 Reviewer
- 可配置触发条件：例如用户输入长度、对话轮数、是否包含新事实

**涉及文件**：
- `modules/reviewer/reviewer_module.py`
- `agent/react_agent.py`

---

## 5. SOUL / USER 文件内容示例

**现状**：`SOUL.md` 和 `USER.md` 为空，只留了标题。  
**目标**：为“单用户 + 单 Agent 的助理/陪伴系统”寻找或设计合适的内容模板。  
**参考方向**：
- 角色设定（名字、性格、说话风格）
- 用户画像（称呼、偏好、目标）
- 边界与价值观

**涉及文件**：
- `modules/prompt/SystemPrompt/SOUL.md`
- `modules/prompt/SystemPrompt/USER.md`

---

## 6. 前端页面接口规划

**现状**：仅命令行交互。  
**目标**：设计可供前端 / WebSocket / HTTP 调用的统一接口，例如：
- 初始化 Agent
- 发送消息
- 获取历史
- 获取/设置模块开关
- 获取/设置工具、技能开关
- 获取/编辑提示词文件

**涉及文件**：
- 新增 `api/` 或 `server/`
- 现有各模块的 bus action 扩展

---

## 7. 补充项

- **配置中心**：把模块开关、工具开关、技能开关统一到一个配置页面可编辑的位置，目前分散在多个 JSON/命令行参数中
- **日志与可观测性**：记录每次 ReAct 循环、工具调用、Reviewer 输出的结构化日志，方便调试
- **错误恢复机制**：工具调用失败、LLM 输出格式错误时的重试与降级策略
- **单元测试覆盖**：为总线、工具解析、Memory 合并、Reviewer JSON 解析等核心逻辑补测试
- **模型切换平滑化**：当前 `ModelScopeLLM` 和 `OpenAILLM` 是两套实现，未来可统一为通过配置切换

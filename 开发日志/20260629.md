## Reviewer 流程

以下顺序图展示了最终答案输出后，后台 Reviewer 模块自动维护 `memory.md` 和 `USER.md` 的过程。该过程对用户不可见。

```mermaid
sequenceDiagram
    autonumber
    actor U as 用户
    participant A as ReactAgent
    participant PM as PromptManager
    participant TM as ToolModule
    participant SM as SkillModule
    participant MM as MemoryModule
    participant LLM as LLM
    participant RM as ReviewerModule
    participant Mem as modules/mem/memory.md
    participant UserFile as modules/prompt/SystemPrompt/USER.md

    U->>A: 用户输入
    A->>PM: get_system_prompt(default)
    PM-->>A: AGENT + SOUL + USER + HEARTBEAT
    A->>TM: format_tool_descriptions
    TM-->>A: 工具说明
    A->>SM: format_skill_overview
    SM-->>A: 技能目录
    A->>MM: get_memory
    MM-->>A: memory.md 内容

    loop ReAct 循环
        A->>LLM: generate(messages)
        LLM-->>A: Thought + Action
        alt Action 是工具调用
            A->>TM: call(tool_name, params)
            TM-->>A: 工具结果
        end
    end

    A->>U: 输出最终答案（用户可见流程结束）

    Note over A,UserFile: 最终答案输出后，后台自动触发 Reviewer

    A->>PM: get_system_prompt(SOUL)
    PM-->>A: SOUL.md
    A->>PM: get_system_prompt(USER)
    PM-->>A: USER.md
    A->>MM: get_memory
    MM-->>A: 当前 memory.md
    A->>PM: get_system_prompt(reviewer)
    PM-->>A: reviewer.md

    A->>A: 拼接 reviewer 完整系统提示词
    A->>RM: review(system_prompt, conversation_text)
    RM->>LLM: generate(reviewer_messages)
    LLM-->>RM: JSON(memory_entries, user_profile_updates, summary)
    RM-->>A: 结构化审查结果

    alt 存在 memory_entries
        A->>Mem: 直接追加写入
    end

    alt 存在 user_profile_updates
        A->>UserFile: 读取 → 合并 → 覆盖写入
    end

    A->>A: _add_to_history(user, assistant)
```

## 模块化Agent架构设计与实现

### 一、架构概述

实现了基于事件总线的模块化Agent架构，所有模块通过总线通信，实现即插即用。

**核心组件：**
- **EventBus（事件总线）**：模块间通信的中枢，实现发布/订阅模式
- **BaseModule（模块基类）**：所有模块的接口定义
- **BaseAgent（Agent基类）**：模块管理和生命周期管理
- **ReactAgent（ReAct Agent）**：实现Thought-Action-Observation循环

### 三、完整调用流程

#### 阶段一：模块注册（用户代码）

```python
# 1. 创建Agent（内部创建EventBus）
agent = ReactAgent(max_iterations=5)
# → BaseAgent.__init__()
#   → self.bus = EventBus()
#   → self._modules = {}
#   → self._enabled = {}

# 2. 创建模块实例（每个模块都持有bus引用）
prompt_mgr = PromptManager("prompt", agent.bus)
# → BaseModule.__init__("prompt", agent.bus)
#   → self.module_id = "prompt"
#   → self.bus = agent.bus

llm = ModelScopeLLM("llm", agent.bus, model_path="./Qwen/Qwen3-0.6B")
# → BaseModule.__init__("llm", agent.bus)
#   → self.module_id = "llm"
#   → self.bus = agent.bus
#   → self.model_path = "./Qwen/Qwen3-0.6B"

tools = BaseTool("tools", agent.bus)
# → BaseModule.__init__("tools", agent.bus)
#   → self.module_id = "tools"
#   → self.bus = agent.bus
#   → self._tools = {}  # 工具字典

# 3. 注册工具函数到ToolModule
tools.register_tool("get_weather", get_weather)
# → self._tools["get_weather"] = get_weather

# 4. 注册模块到Agent
agent.register_module(prompt_mgr, enabled=True)
# → self._modules["prompt"] = prompt_mgr
# → self._enabled["prompt"] = True

agent.register_module(llm, enabled=True)
# → self._modules["llm"] = llm
# → self._enabled["llm"] = True

agent.register_module(tools, enabled=True)
# → self._modules["tools"] = tools
# → self._enabled["tools"] = True
```

#### 阶段二：Agent启动

```python
agent.start()
# → BaseAgent.start()
#   → 遍历 self._modules
#   → 对于每个启用的模块：
#     → module.initialize()
#       → prompt_mgr.initialize()  # 扫描SystemPrompt/*.md，加载到self._prompts
#       → llm.initialize()         # 加载模型和分词器
#       → tools.initialize()       # 当前为空
#     → module.register()
#       → prompt_mgr.register()
#         → self.bus.subscribe("prompt", self.handle_message)
#           → bus._subscribers["prompt"].append(prompt_mgr.handle_message)
#       → llm.register()
#         → self.bus.subscribe("llm", self.handle_message)
#           → bus._subscribers["llm"].append(llm.handle_message)
#       → tools.register()
#         → self.bus.subscribe("tools", self.handle_message)
#           → bus._subscribers["tools"].append(tools.handle_message)
```

此时 `bus._subscribers` 内容：
```python
{
    "prompt": [prompt_mgr.handle_message],
    "llm": [llm.handle_message],
    "tools": [tools.handle_message]
}
```

**步骤2：通过总线获取系统提示词**

#                    payload={'action': 'get_system_prompt', 'prompt_name': 'travel'}, timeout=5.0)

#           → self.send(target='agent', payload={'system_prompt': system_prompt},

#     → bus._subscribers.clear()

### 四、关键设计点

1. **纯同步通信**：`bus.request()` 是同步的，`publish()` 内部同步调用 handler，handler 同步处理后回复，回复被存入 `_responses` 字典，`publish()` 返回后直接取出。

2. **correlation_id 匹配**：每个请求生成唯一 `correlation_id`，响应时原样带回，用于匹配请求和响应。

3. **定向投递**：`message.target` 指定目标模块，总线只投递给目标模块，不广播。

4. **模块解耦**：Agent 不直接调用模块方法，所有交互都通过总线。模块之间互不依赖，可以即插即用。

### 五、使用示例

```python
from bus import EventBus
from agent import ReactAgent
from modules.prompt import PromptManager
from modules.llm import ModelScopeLLM
from modules.tools import BaseTool
from modules.tools.system_tools.weather import get_weather

# 创建Agent
agent = ReactAgent(max_iterations=5)

# 创建模块
prompt_mgr = PromptManager("prompt", agent.bus)
llm = ModelScopeLLM("llm", agent.bus, model_path="./Qwen/Qwen3-0.6B")
tools = BaseTool("tools", agent.bus)

# 注册工具
tools.register_tool("get_weather", get_weather)

# 注册模块
agent.register_module(prompt_mgr, enabled=True)
agent.register_module(llm, enabled=True)
agent.register_module(tools, enabled=True)

# 启动Agent
agent.start()

# 运行Agent
result = agent.run("查苏州天气", prompt_name="travel")

# 停止Agent
agent.stop()
```

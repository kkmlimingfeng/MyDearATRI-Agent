## 输出格式要求

你的每次回复必须严格遵循以下格式，包含一个 Thought 和至少一个 Action：

Thought: [你的思考过程和下一步计划]
Action: [你要执行的具体行动]

Action的格式必须是以下之一：
1. 调用工具：工具名(参数名="参数值")。
2. 结束任务：Finish[你的最终答案内容]。

## 多 Action 输出

当多个工具调用相互独立、不需要等待彼此结果时，你可以在一次回复中输出多个 Action，系统会在同一轮内按顺序执行，并把结果合并返回给你：

```
Thought: 北京和上海的天气查询相互独立，可以一起执行。
Action: get_weather(city="北京")
Action: get_weather(city="上海")
```

系统返回的 Observation 格式如下：

```
Observation:
- get_weather(city="北京") -> 北京当前天气:晴，气温27摄氏度
- get_weather(city="上海") -> 上海当前天气:阴，气温25摄氏度
```

收到多个结果并且任务已完成后，你必须**立即总结**并输出 `Action: Finish[你的最终答案]`，不要再次调用相同工具。

注意事项：
- 仅当多个工具调用**完全独立**时才一起输出。
- 如果一个工具的结果依赖另一个工具的输出，请分轮次调用。
- `Finish[...]` 不能与其他 Action 同时出现，必须是单独一个 Action。

## 重要提示

- 每次只输出一个 Thought，但可以输出一个或多个 Action。
- 可用工具是以函数调用格式提供的，调用工具时必须严格按照函数调用格式进行。
- 每个 Action 必须在同一行，不要换行。
- 如果用户的问题**不需要使用任何工具或技能**（例如打招呼、闲聊、问“你好”），你必须直接回答，格式为：
  `Thought: 用户的问题不需要使用工具，直接回答即可。 Action: Finish[你的友好回答]`
- 仅当收集到足够信息可以回答用户问题时，必须使用 `Action: Finish[你的最终答案内容]` 格式结束。
- **严禁循环调用同一个工具**：如果你已经获得过某个工具的返回结果，请直接基于该结果回答，禁止以相同参数再次调用。
